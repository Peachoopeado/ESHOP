Manage.py - файл, с которым больше всего нужно работать:
1) Запускается локальный сервер;
2) Создаются файлы;
3) проводятся миграции.

__init__.py - файл, в который выписываются характеристики, которые обрабатываются при запуске проекта

asgi.py и wsgi - обеспечивают корректное подключение к серверу. При выгрузке на удалённый сервер происходит выгрузка
сайта.

settings.py - в нём описываются все глобальные настройки для Django проекта.
Переменные:
BASE_DIR - полный путь к проекту.

SECRET_KEY - код безопасности сайта. Перед выкатыванием нужно поменять этот ключ.

DEBUG = True/False - отображение ошибок на страницах сайта. После выгрузки поставить False.

ALLOWED_HOSTS = []

INSTALLED_APPS - набор всех установленных в проекте приложений (встроенные и созданные)

MIDDLEWARE - Плагины и библиотек, которые обеспечивают безпасности внутри проекта, работу сессии, поддержку csrf токена,
авторизацию

ROOT_URLCONF - основной файл ссылок

TEMPLATES - шаблоны

WSGI_APPLICATION

DATABASES - базы данных, с которыми ведётся работа





urls.py - файл, в котором отслеживаются url-адреса. При переходе по определенным адресам будет открываться та или иная
страница.



migrations - нужен для описания структуры базы данных

admin.py - в этом файле указываются, какие таблицы должны отображаться в панели администратора


apps.py - настройки конкретного приложения

models - в этом файле создаётся класс, на основе которого создаётся таблица в базе данных.

tests.py - выполняется unit-тестирование

views.py - отвечает за те методы, которые будут вызваны при переходе пользователем на определенную страницу.
